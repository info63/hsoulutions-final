## üåê OFFLINE MESH COMMUNICATION: HOW EVERYONE CONNECTS

### **The Problem:**
Traditional apps need servers. Servers =
- Privacy risk
- Single point of failure
- Cost to operate
- Requires internet

### **The Solution: Peer-to-Peer Mesh**

**How it works:**

```
Shannon's Phone ‚Üê‚Üí Crystal's Phone ‚Üê‚Üí Liv's Phone
      ‚Üï                  ‚Üï                 ‚Üï
  Neighbor         Friend           Local Liv User
      ‚Üï                  ‚Üï                 ‚Üï
  [Creates offline network with NO central server]
```

**Technical:**
- Uses Bluetooth Low Energy + WiFi Direct
- Each phone becomes a node
- Messages hop from phone to phone until reaching recipient
- End-to-end encrypted (each message has unique key)
- No central server (can't be shut down, can't be surveilled)

